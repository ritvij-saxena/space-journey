## Directory Purposes

**frontend/:**
- Purpose: Web application frontend serving the generative art installation
- Contains: JavaScript modules, HTML, CSS, static assets
- Key files: `main.js` (entry point), `package.json` (dependencies)

**frontend/src/:**
- Purpose: Source code for all application logic
- Contains: ES modules (classes, utilities), shader definitions, styles
- Key files: `main.js`, `wasm-loader.js`, `particle-renderer.js`, `weather.js`

**frontend/src/shaders/:**
- Purpose: Custom shader definitions for post-processing effects
- Contains: JavaScript objects exporting GLSL shader code
- Key files: `chromatic-aberration.js`

**frontend/public/:**
- Purpose: Static assets copied directly to output without processing
- Contains: Pre-generated textures, audio files
- Key files: `textures/sd_atlas.png` (Stable Diffusion texture atlas)

**frontend/dist/:**
- Purpose: Built application output (generated)
- Contains: Minified JavaScript, bundled assets, HTML
- Generated by: `npm run build` (Vite bundler)
- Not committed: Rebuild required after pulling changes

**rust-wasm/src/:**
- Purpose: Rust source code for WASM module
- Contains: Single `lib.rs` file with all logic (NoiseField implementation)
- Key types: `NoiseField` struct, `Perlin` noise wrapper, `Fbm` fractal Brownian motion

**rust-wasm/pkg/:**
- Purpose: Compiled WASM bindings (generated)
- Contains: `.wasm` binary, JavaScript wrapper (*.js), TypeScript definitions (*.d.ts)
- Generated by: `wasm-pack build --target web`
- Imported as: `import("unsupervised-wasm/unsupervised_wasm.js")`

## Key File Locations

**Entry Points:**
- `frontend/index.html`: HTML document, loads YouTube API, imports main.js ES module
- `frontend/src/main.js`: JavaScript entry point, instantiates UnsupervisedApp class
- `rust-wasm/src/lib.rs`: Rust entry point, defines NoiseField interface for WASM

**Configuration:**
- `frontend/package.json`: npm scripts (dev, build, wasm:build, build:all), dependencies (three, axios)
- `frontend/vite.config.js`: Vite bundler config (plugins, WASM alias, dev server, build output)
- `rust-wasm/Cargo.toml`: Rust dependencies (wasm-bindgen, noise, web-sys), release optimizations

**Core Logic:**
- `frontend/src/main.js`: UnsupervisedApp class (scene setup, animation loop, event handling)
- `frontend/src/wasm-loader.js`: WASM module singleton, function wrappers (initWasm, createNoiseField, generateParticles)
- `frontend/src/particle-renderer.js`: ParticleRenderer class (geometry, materials, update logic)
- `frontend/src/weather.js`: WeatherManager class (procedural parameters, interpolation, device motion)
- `rust-wasm/src/lib.rs`: NoiseField struct (noise generation, particle computation, weather mapping)

**Visual Effects:**
- `frontend/src/post-processing.js`: PostProcessor class (EffectComposer, bloom, chromatic aberration)
- `frontend/src/shaders/chromatic-aberration.js`: Shader material for radial RGB split effect
- `frontend/src/raymarcher.js`: Raymarcher class (alternative rendering approach, reference implementation)

**Styling:**
- `frontend/src/styles.css`: UI layout and appearance (info panel, overlay, audio hint)

**Static Assets:**
- `frontend/public/textures/`: Pre-generated texture atlases for art rendering
- `frontend/public/audio/`: Audio files (if used as fallback)

## Naming Conventions

**Files:**
- Source modules: camelCase with .js extension (`particleRenderer.js`, `wasmLoader.js`)
- Exported classes: PascalCase matching filename (`class ParticleRenderer` in `particle-renderer.js`)
- Shaders: Kebab-case descriptive names (`chromatic-aberration.js`)
- Built output: Hash-suffixed for cache busting (`index-DuOSS43Q.css`)
- Cargo projects: Snake_case (`unsupervised-wasm`, `unsupervised_wasm.js` in pkg)

**Directories:**
- Source: `src/`
- Build output: `dist/`
- Dependencies: `node_modules/` (JS), `target/` (Rust)
- Configuration: Root level (`package.json`, `vite.config.js`, `Cargo.toml`)
- Asset subdirectories: Plural descriptive (`public/textures/`, `public/audio/`)

**JavaScript Classes:**
- PascalCase for class names: `UnsupervisedApp`, `ParticleRenderer`, `WeatherManager`, `PostProcessor`
- Lowercase for instances: `this.particleRenderer`, `this.weatherManager`
- Private fields use underscore prefix: `this._canvas`, `this._textureDrawn`

**Variables & Functions:**
- camelCase: `getDelta()`, `getCurrentWeather()`, `particleCount`, `wasmModule`
- Constants: UPPER_SNAKE_CASE: `PARTICLE_COUNT = 15000`
- Boolean flags: is/has prefix: `isWasmReady`, `hasLoaded`

**Rust:**
- Modules/structs: PascalCase: `NoiseField`
- Methods: snake_case: `generate_particles()`, `weather_to_params()`
- Fields: snake_case: `time`, `fbm`

## Where to Add New Code

**New Rendering Feature (particles, effects):**
- Primary code: `frontend/src/` → create new class (e.g., `particle-emitter.js`)
- Integrate into: `frontend/src/main.js` UnsupervisedApp.init()
- Add to scene: `this.scene.add(emitter.getMesh())`
- Pattern: Follow ParticleRenderer structure (init → getMesh → update → dispose)

**New Post-Processing Effect:**
- Shader definition: `frontend/src/shaders/` → new .js file exporting shader object
- Effect wrapper: `frontend/src/post-processing.js` → add new pass to composer
- Pattern: Follow ChromaticAberrationShader (uniforms, vertexShader, fragmentShader)

**New WASM Computation:**
- Implementation: `rust-wasm/src/lib.rs` → add new #[wasm_bindgen] function/method
- JavaScript wrapper: `frontend/src/wasm-loader.js` → export function calling WASM
- Rebuild: `npm run wasm:build` or use monorepo script `npm run build:all`

**New UI Component:**
- HTML: `frontend/index.html` → add element with clear ID
- Styling: `frontend/src/styles.css` → add class rules
- Logic: `frontend/src/main.js` → query element, attach listeners, update during animate()

**New Utility/Helper:**
- Module: `frontend/src/` → create new .js file exporting functions
- Export: Named exports preferred for tree-shaking: `export function helperName() {}`
- Import: Explicit imports in consuming modules: `import { helperName } from "./helper.js"`

**Unit Testing (if added):**
- Test files: Same directory as source with `.test.js` or `.spec.js` suffix
- Location: `frontend/src/__tests__/` or co-locate (e.g., `weather.test.js`)
- Pattern: Jest or Vitest (not currently configured)

## Special Directories

**frontend/dist/:**
- Purpose: Build output from Vite
- Generated: Yes (via `npm run build`)
- Committed: No (included in .gitignore, rebuild from source)
- Contents: Minified JS/CSS, hashed asset filenames, sourcemaps (in dev builds)

**rust-wasm/target/:**
- Purpose: Rust build artifacts (intermediate objects, deps)
- Generated: Yes (via `cargo build`, `wasm-pack build`)
- Committed: No (included in .gitignore)
- Size: Large (hundreds of MB), rebuild takes 30-60 seconds

**rust-wasm/pkg/:**
- Purpose: Compiled WASM output and JavaScript bindings
- Generated: Yes (via `wasm-pack build --target web`)
- Committed: No (best practice: rebuild from source for reproducibility)
- Contains: `.wasm` binary, JavaScript wrapper, type definitions
- Aliased in Vite: `unsupervised-wasm` resolves to this directory

**frontend/node_modules/:**
- Purpose: Installed npm dependencies
- Generated: Yes (via `npm install`)
- Committed: No (use package.json/package-lock.json)
- Size: Large (hundreds of MB), install takes 1-2 minutes on clean install

**frontend/.vite/ (inside node_modules):**
- Purpose: Vite dev server cache
- Generated: Yes (during `npm run dev`)
- Committed: No (cleaned by `npm run clean`)

## Build & Deployment Files

**frontend/vite.config.js:**
- Purpose: Vite bundler configuration
- Key settings:
  - base: `/unsupervised_moma_replica/` for GitHub Pages
  - Plugins: wasm, topLevelAwait, glsl
  - Alias: `unsupervised-wasm` → `../rust-wasm/pkg`
  - Output: Hash-based filenames, Terser minification

**frontend/package.json:**
- Purpose: npm project metadata and scripts
- Key scripts:
  - `dev`: Start Vite dev server on port 3000
  - `build`: Build for production (outputs to dist/)
  - `wasm:build`: Rebuild Rust/WASM module
  - `build:all`: Full build (WASM first, then frontend)

**rust-wasm/Cargo.toml:**
- Purpose: Rust project metadata and dependencies
- Key dependencies:
  - wasm-bindgen 0.2: JavaScript FFI
  - noise 0.9: Perlin/FBM noise
  - web-sys 0.3: Browser APIs (console, canvas, etc.)
- Release optimizations: opt-level = "z" (minimize size), lto = true

## Development Workflow

**First-time setup:**
```bash
cd frontend
npm install
cd ../rust-wasm
wasm-pack build --target web --release
cd ../frontend
npm run dev  # Runs dev server on localhost:3000
```

**Making code changes:**
- JavaScript files: Changes auto-reload via Vite HMR
- Rust/WASM: Require manual rebuild via `npm run wasm:build`

**Building for production:**
```bash
npm run build:all  # Builds WASM + frontend together
# Output: frontend/dist/ (deployable)
```

**Deployment (GitHub Pages):**
```bash
# Push dist/ to gh-pages branch or use GitHub Actions
# Note: base path in vite.config.js assumes repo name 'unsupervised_moma_replica'
```
